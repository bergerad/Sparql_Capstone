What is SPARQL?

SPARQL is a recursive acryonym which stands for "SPARQL Protocol and RDF Query Language." This query language is specially designed for for returning and manipulating RDF triples from databases like DBpedia. Queries return tables which can then be structured as XML, JSON, CSV or TSV so that the results can be used in applications. The SPARQL Overview document on W3.org (link can be found in the resource sub-section) states that:

SPARQL can be used to express queries across diverse data sources, whether the data is stored natively as RDF or viewed as RDF via middleware. SPARQL contains capabilities for querying required and optional graph patterns along with their conjunctions and disjunctions. SPARQL also supports extensible value testing and constraining queries by source RDF graph.

The syntax and styling of SPARQL is based on SQL and should be easy to follow if you are already familiar with SQL. In the following sub-sections you will find a very quick and basic introduction to SPARQL syntax which you will encounter in 'SPARQL Queries' and 'More SPARQL Queries' as well as some resources which you might find helpful when learning about and using SPARQL.

Quick, Basic Introduction to Structure of SPARQL Queries

In this tutorial, we will be focusing mainly on SELECT queries which returns a table. Since SPARQL deals with triples, the WHERE clause requires that its body matches a triple pattern. Note, we can think of triples as typically following the general pattern: subject, property, object.

Some things to note before we get into the general structure is that you may match triples by using variables. Variables are prefixed by a question mark ?.
You will also need to specify URIs for most of the triple patterns as well. URIs must be contained within angled brackets <,>.

General Structure

Below is a general structure of a SELECT query that matches exactly one variable (as the subject). 

SELECT ?desiredVariable
WHERE 
{
   ?desiredVariable <URI_for_property> <URI_for_object>
}

In addition to URIs, you may use quotes to in order to match a specific value (case sensitive): 

   ?desiredVariable "specific property" <URI for object>

To simplify the queries with repeated URIs which differ only by the selected feature, you can use the PREFIX attribute. 

PREFIX foo <URI_for_database#prefix>
SELECT ?desiredVariable
WHERE
{
   ?desiredVariable foo:bar foo:baz
}
